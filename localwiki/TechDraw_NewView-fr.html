Description

L'outil Vue ajoute une représentation d'un ou plusieurs objets à une page de dessin. C'est le bloc de construction de base du module TechDraw. La plupart des autres vues sont dérivées d'une manière ou d'une autre de NewView.

Comment faire

  1. Sélectionnez un ou plusieurs objets (Corps, App::Part, Part::Feature, Draft object, ... Voir Notes) dans la fenêtre ou l'arbre 3D.
  2. Si vous avez plusieurs pages de dessin dans votre document, vous devrez également sélectionner la page souhaitée dans l'arborescence.
  3. Appuyez sur le bouton Vue

Options

Aucune.

Propriétés

  • DONNÉESX: La position horizontale de la vue sur la page. (1)
  • DONNÉESY: La position verticale de la vue sur la page. (1)
  • DONNÉESLockPosition: Empêche les vues d'être glissées dans le Gui quand True. La vue peut toujours être déplacée en changeant les propriétés X, Y. (1)
  • DONNÉESRotation: Rotation antihoraire de la vue sur la page en degrés. (1)
  • DONNÉESScaleType: "Document": utilise le paramètre d'échelle de la page. "Custom": utilise une échelle unique pour cette vue. "Automatic": ajuste la vue à la page. (1)
  • DONNÉESScale: Une vue sera rendue sur la page à l'Échelle: 1 par rapport à la Source. (1)
  • DONNÉESCaption: Légende de texte court facultative.
  • DONNÉESSource: Liens vers les objets projetables à représenter
  • DONNÉESDirection: Un vecteur représentant la direction de visualisation. Les directions communes sont Vue de face: (0,-1,0), Vue de dessus: (0,0,1), Vue de droite: (1,0,0), Vue Axonométrique: (1,1,1). (1)
  • DONNÉESPerspective: Vrai pour la projection en perspective, faux pour la projection orthogonale.
  • DONNÉESFocus: Distance de la caméra au plan de projection pour les projections en perspective. Doit être ajusté pour s'adapter à l'objet. Trop loin et la perspective est perdue, trop proche et l'objet est déformé.
  • DONNÉESCoarseView: Si true, TechDraw utilisera une approximation de polygone pour calculer la géométrie de dessin. Si faux, TechDraw utilisera un algorithme de précision. Voir les notes.
  • DONNÉESSmooth Visible Lines: Lignes lisses visibles sont activées / désactivées.
  • DONNÉESSeam Visible Lines: Les lignes de couture visibles sont activées / désactivées.
  • DONNÉESIso Visible Lines: Les lignes isométriques visibles (u, v) sont activées / désactivées.
  • DONNÉESHard Hidden Lines: Lignes cachées activées / désactivées.
  • DONNÉESSmooth Hidden Lines: Lignes lisses cachées activées / désactivées.
  • DONNÉESSeam Hidden Lines: Lignes de couture cachées activées / désactivées.
  • DONNÉESIso Hidden Lines: Les lignes isométriques cachées (u, v) sont activées / désactivées.
  • DONNÉESIso Count: Nombre de lignes isométriques (u, v) à dessiner sur chaque face.
  • VUEKeep Label: Toujours afficher l'étiquette de vue si elle est vraie.
  • VUELineWidth: L'épaisseur des lignes visibles. Voir Groupes de lignes.
  • VUEHiddenWidth: L'épaisseur des lignes cachées, si activée.
  • VUEIsoWidth: L'épaisseur des lignes de surface isométriques (u, v) et des lignes de cotes.
  • VUEExtraWidth: pas encore mis en œuvre.
  • VUEShowCenters: Les marques des centres de cercle / d'arc sont activées / désactivées.
  • VUECenterScale: Ajustement de la taille du repère du centre de l'arc circulaire, si activé.
  • VUEHorizCenterLine: Afficher une ligne centrale horizontale à travers la vue.
  • VUEVertCenterLine: Afficher une ligne centrale verticale à travers la vue.
  • VUEShowSectionLine: Afficher / masquer la ligne de section, le cas échéant.

(1) ces propriétés sont communes à tous les types de vue.

Script

Les vues peuvent être ajoutées aux Pages en utilisant le code Python. 

view = FreeCAD.ActiveDocument.addObject('TechDraw::DrawViewPart','View')
rc = page.addView(view)
FreeCAD.ActiveDocument.View.Source = [App.ActiveDocument.Box]
FreeCAD.ActiveDocument.View.Direction = (0.0,0.0,1.0)

Notes

  • La vue dessine n'importe quoi avec une propriété de forme. Vous pouvez également sélectionner des objets brouillons ou des corps. View extraira également toutes les formes des objets dans un conteneur App :: Part ou un groupe DocumentObject.
  • CoarseView peut être beaucoup plus rapide pour les modèles complexes. La qualité du dessin est réduite car chaque courbe est approximée par une série de segments de ligne courts. Les sommets ne sont pas affichés dans CoarseView, car chaque segment court produirait deux nouveaux sommets et l'affichage deviendrait encombré. Les dimensions linéaires peuvent être ajoutées à un CoarseView, mais elles ne sont probablement pas utiles.
  • Notez que CoarseView est affecté par un bogue en amont dans OCCT (#3332) ce qui fait que la position de la vue sur la page est légèrement décalée par rapport aux valeurs X, Y spécifiées.
